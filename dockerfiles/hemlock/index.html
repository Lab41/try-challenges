Hemlock Demo

Hemlock quick start (Click 'Start Hemlock' in the top right):

  • Load the data by performing 5 simple steps.
  • Once the data is loaded, the link for Kibana below will automatically be populated with the results.
  • At any time you can type hemlock or hemlock COMMAND to get additional information

  • Step 1: Store hemlock credentials.
    • hemlock hemlock-server-store --credential_file /src/hemlock_creds
      • 
      
    
      
  • Step 2: Create a tenant (project) for a system with data to live in.
    • 
      
      
        
    • hemlock tenant-create --name fileserver_tenant
      • 
      
    
      
  • Step 3: Register a system that belongs to that tenant.
    • 
      
      
        
    • Copy the UUID that was output from creating the tenant and use it for the --tenant_id field. For example:
      • 
        
    • hemlock register-local-system --name System1 --data_type files --description "description" \
                              --endpoint http://hemlock.server/ --poc_name user1 --poc_email user1@email.com \
                              --hostname system1.fqdn --tenant_id 7d0f6b0d-334a-4d89-bd1a-70e8e1c04aa6
      • 
      
    
      
  • Step 4: Store client credentials.
    • 
      
      
        
    • Copy the UUID that was output from registering the system and use it for the --system_id field. For example:
      • 
        
    • hemlock client-store --name fs_client_1 --type fs --credential_file /src/fs_creds --system_id 7d0f6b0d-334a-4d89-bd1a-70e8e1c04aa6
      • 
      
    
      
  • Step 5: Run the client and load the data.
    • 
      
      
        
    • Copy the UUID that was output from storing the client and use it for the --uuid field. For example:
      • 
        
    • hemlock client-run --uuid 7d0f6b0d-334a-4d89-bd1a-70e8e1c04aa6
      • 
      
    
    

    
The data provided for this demo is synthetic in combination with datasets from IEEE Visual Analytics Science and Technology (VAST) Challenges.
